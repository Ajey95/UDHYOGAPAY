\section*{Project Overview}

\textbf{Udhyoga Pay} is a comprehensive hyperlocal marketplace platform designed to seamlessly connect service workers with users seeking various services. The platform leverages modern web technologies and AI-driven algorithms to provide real-time service matching, secure payment processing, and transparent review systems.

\subsection*{My Role: User \& Matching Management Module}
I implemented the complete demand-side of the Udhyogapay marketplace, covering customer identity and authentication, geospatial worker discovery using MongoDB's \texttt{\$near} geospatial query operator against a 2dsphere-indexed \texttt{workers.currentLocation} field, AI-assisted multi-factor worker ranking, and a structured post-job review system with a MongoDB aggregation post-save hook that automatically recalculates a worker's average rating.

\subsection*{Module Components}
\begin{itemize}[leftmargin=*]
    \item \textbf{Submodules (5):} Auth \& Profile, Geospatial Discovery, Review System, Saved Address Management, Notification \& Session Management
    \item \textbf{Master Tables (2):} users, savedaddresses
    \item \textbf{Transaction Tables (4):} reviews, notifications, searchhistory, usersessions
    \item \textbf{AI Integration:} Matching Engine microservice (port 5002) with urgency/skill/rating/distance multi-factor scoring with Haversine fallback
\end{itemize}

\section*{Detailed Contribution Analysis}

\subsection*{1. Node.js Controllers \& Business Logic (10 Marks)}
\begin{itemize}[leftmargin=*]
    \item \textbf{authController.ts:} User registration with bcrypt.hash(password, 12), JWT issuance (7-day expiry), login validation, password reset token generation using crypto.randomBytes(20) + SHA-256 hashing
    \item \textbf{matchingController.ts:} Geospatial worker discovery using MongoDB \texttt{\$near} operator, AI engine HTTP call with axios.post + 5-second timeout, automatic Haversine fallback on AI unavailability
    \item \textbf{reviewController.ts:} Review submission with Review.post('save') aggregation hook that recalculates workers.rating via \texttt{\$group: \{ avgRating: \{ \$avg: '\$rating' \} \}}
    \item \textbf{savedAddressController.ts:} Max 5 addresses per user enforcement, pre-save hook to clear previous isDefault=true when new default is set
    \item \textbf{userController.ts:} Profile CRUD with JWT-protected routes, location updates with 2dsphere index queries
\end{itemize}

\subsection*{2. Express.js Routes \& Middleware (10 Marks)}
\begin{itemize}[leftmargin=*]
    \item \textbf{auth.ts (7 routes):} POST /register, POST /login, GET /logout, POST /forgot-password, POST /reset-password/:token, GET /profile, PATCH /update-password
    \item \textbf{users.ts (6 routes):} GET /profile, PATCH /profile, GET /saved-addresses, POST /saved-addresses, PATCH /saved-addresses/:id, DELETE /saved-addresses/:id
    \item \textbf{matching.ts (2 routes):} POST /find (AI-assisted geospatial search), GET /search-history
    \item \textbf{reviews.ts (7 routes):} POST / (create), GET /worker/:workerId, GET /booking/:bookingId, PATCH /:id, DELETE /:id, GET /my-reviews, GET /admin/pending
    \item \textbf{Middleware Chain:} protect (JWT validation from httpOnly cookie), isUser, isWorker, isAdmin role guards
\end{itemize}

\subsection*{3. MongoDB Schemas \& Indexes (5 Marks)}
\begin{itemize}[leftmargin=*]
    \item \textbf{users schema:} 2dsphere index on location.coordinates, unique indexes on email and phone, pre-save bcrypt hook, getResetPasswordToken() instance method
    \item \textbf{savedaddresses schema:} Compound index \{userId: 1, label: 1\}, pre-save hook enforcing single isDefault=true per user
    \item \textbf{reviews schema:} Compound index \{worker: 1, status: 1\}, unique index on booking field, post-save aggregation pipeline for live rating calculation
    \item \textbf{notifications schema:} Index \{userId: 1, createdAt: -1\}, referenceId FK to bookings/payouts
    \item \textbf{searchhistory schema:} Index \{userId: 1, createdAt: -1\}, tracks profession, coordinates, aiEngineUsed boolean
\end{itemize}

\subsection*{4. Third-Party Service Integration (5 Marks)}
\begin{itemize}[leftmargin=*]
    \item \textbf{AI Matching Engine (Port 5002):} axios.post with 5-second timeout, sends serviceType, userLocation, urgency, maxDistance, minRating, skills array; receives ranked worker list with matchScore
    \item \textbf{Nominatim Geocoding:} osmService.geocodeAddress() converts text address to lat/lon, reverseGeocode() converts coordinates to human-readable address
    \item \textbf{OSRM Routing:} osmService.getRoute() returns GeoJSON LineString for map route overlay from user to selected worker
    \item \textbf{Fallback Mechanism:} On AI engine HTTP error, Haversine distance formula auto-activates: distance = 2 × R × arcsin(√[sin²(Δlat/2) + cos(lat1)×cos(lat2)×sin²(Δlon/2)])
\end{itemize}

\subsection*{5. AI Matching Algorithm \& Innovation (5 Marks)}
\begin{itemize}[leftmargin=*]
    \item \textbf{Multi-Factor Scoring (AI Engine):}
    \begin{itemize}
        \item Distance score: (1 - distance / maxDistance) × 100
        \item Urgency multiplier: Low=1.0, Medium=1.2, High=1.5
        \item Rating weight: (worker.rating / 5) × 20
        \item Skill match: skills.filter(s ⇒ worker.skills.includes(s)).length × 10
        \item Final matchScore = (distanceScore + ratingWeight + skillMatch) × urgencyMultiplier
    \end{itemize}
    \item \textbf{Haversine Fallback:} Pure distance-based ranking when AI unavailable
    \item \textbf{Innovation Highlights:}
    \begin{itemize}
        \item Saved address book with pre-save single-default enforcement
        \item Review.post('save') aggregation pipeline for live worker rating updates
        \item Max-5 saved addresses per user with validation
    \end{itemize}
\end{itemize}

\section*{API Endpoints Documentation}

\subsection*{API 1: POST /api/matching/find}
\begin{itemize}[leftmargin=*]
    \item \textbf{Purpose:} Finds all online, verified workers matching requested profession. Passes data to AI engine for scoring, or uses Haversine fallback
    \item \textbf{Request:} \texttt{\{ "serviceType": "plumber", "userLocation": \{ "latitude": 14.6819, "longitude": 77.5931 \}, "urgency": "high", "maxDistance": 25, "minRating": 3.5, "skills": [] \}}
    \item \textbf{MongoDB Query:} Worker.find(\{ profession: new RegExp(serviceType, 'i'), isOnline: true, isVerified: true, 'currentLocation.coordinates.0': \{ \$ne: 0 \} \}).populate('userId', 'name email phone profileImage')
    \item \textbf{AI Engine Fallback:} matchScore = distance <= maxDistance ? (1 - distance / maxDistance) × 100 : 0; estimatedTime = Math.ceil((distance / 30) × 60)
    \item \textbf{Response:} \texttt{\{ "data": \{ "workers": [...], "totalMatches": 5, "searchRadius": 25, "matchingAlgorithm": "Fallback-Distance-Based" \} \}}
    \item \textbf{Auth:} JWT protected (protect middleware)
\end{itemize}

\subsection*{API 2: POST /api/users/saved-addresses}
\begin{itemize}[leftmargin=*]
    \item \textbf{Purpose:} Creates saved address for authenticated user. Enforces max 5 addresses. Auto-clears prior isDefault=true if new entry is marked default
    \item \textbf{Request:} \texttt{\{ "label": "home", "address": "12 MG Road", "coordinates": [77.59, 14.68], "isDefault": true \}}
    \item \textbf{Processing:} const count = await SavedAddress.countDocuments(\{ userId: req.user.\_id \}); if (count >= 5) return 400; if (req.body.isDefault) await SavedAddress.updateMany(\{ userId: req.user.\_id \}, \{ isDefault: false \})
    \item \textbf{Auth:} protect + isUser
\end{itemize}

\section*{Key Technical Achievements}

\begin{enumerate}[leftmargin=*]
    \item \textbf{AI-Assisted Geospatial Matching:} Multi-factor scoring with urgency/skill/rating/distance weighting + automatic Haversine fallback
    \item \textbf{Live Rating Recalculation:} Review.post('save') aggregation hook auto-updates workers.rating on every approved review
    \item \textbf{Saved Address Default Enforcement:} Pre-save hook ensures single isDefault=true per user across all saved addresses
