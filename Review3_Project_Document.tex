\section*{Project Overview}

\textbf{Udhyoga Pay} is a comprehensive hyperlocal marketplace platform designed to seamlessly connect service workers with users seeking various services. The platform leverages modern web technologies and AI-driven algorithms to provide real-time service matching, secure payment processing, and transparent review systems.

\subsection*{Key Features}
\begin{itemize}[leftmargin=*]
    \item \textbf{Real-time Service Matching:} AI-powered algorithm matches users with the most suitable service providers based on location, availability, ratings, and service requirements.
    \item \textbf{Secure Payment Integration:} Escrow-like payment flow ensuring both parties are protected throughout the transaction lifecycle.
    \item \textbf{Geolocation-Based Discovery:} Hyperlocal filtering allows users to find service providers within their immediate vicinity.
    \item \textbf{Comprehensive KYC System:} Robust verification process for service workers ensuring safety and trust.
    \item \textbf{Rating and Review System:} Transparent feedback mechanism building trust within the platform ecosystem.
\end{itemize}

\subsection*{Technology Stack}
\begin{itemize}[leftmargin=*]
    \item \textbf{Frontend:} React, TypeScript, Tailwind CSS, Vite
    \item \textbf{Backend:} Node.js, Express.js, TypeScript
    \item \textbf{Database:} MongoDB (NoSQL)
    \item \textbf{Mobile:} React Native, Expo
    \item \textbf{AI Engine:} Custom TypeScript-based matching algorithm
    \item \textbf{Cloud Services:} Cloudinary (media), Razorpay/Stripe (payments)
    \item \textbf{Authentication:} JWT-based secure authentication
\end{itemize}

\subsection*{Module Details}

\subsubsection*{1. Authentication Module}
Handles user registration, login, JWT token management, password reset functionality, and role-based access control (User, Worker, Admin).

\subsubsection*{2. User/Client Module}
Manages user profiles, service search and discovery, booking creation and management, payment processing, and review submission capabilities.

\subsubsection*{3. Service Provider/Worker Module}
Facilitates worker registration with KYC verification, profile management with portfolio uploads, booking acceptance/rejection, earnings tracking, and availability management.

\subsection*{Database Schema Highlights}

\textbf{Master Tables:}
\begin{itemize}[leftmargin=*]
    \item \textit{Users:} Stores customer information, authentication credentials, location data, preferences
    \item \textit{Workers:} Contains service provider details, KYC documents, skills, ratings, availability
    \item \textit{Services:} Defines available services with descriptions, pricing guidelines, categories
    \item \textit{Categories:} Organizes services into hierarchical categories for easy navigation
\end{itemize}

\textbf{Transaction Tables:}
\begin{itemize}[leftmargin=*]
    \item \textit{Bookings:} Records service bookings with status tracking, timestamps, location data
    \item \textit{Payments:} Manages payment transactions, refunds, escrow states, payment gateway metadata
    \item \textit{Reviews:} Stores ratings and feedback from users about completed services
\end{itemize}

The Udhyoga Pay platform follows a modern microservices-inspired architecture with clear separation of concerns:

\begin{enumerate}[leftmargin=*]
    \item \textbf{Presentation Layer:}
    \begin{itemize}
        \item Web Frontend (React + TypeScript)
        \item Mobile Application (React Native + Expo)
    \end{itemize}
    
    \item \textbf{API Gateway Layer:}
    \begin{itemize}
        \item Express.js REST API
        \item JWT-based authentication middleware
        \item Rate limiting and security middleware
    \end{itemize}
    
    \item \textbf{Business Logic Layer:}
    \begin{itemize}
        \item Authentication service
        \item Booking management service
        \item Payment processing service
        \item Matching algorithm engine
        \item Notification service
    \end{itemize}
    
    \item \textbf{Data Layer:}
    \begin{itemize}

        \item Redis cache (planned)
        \item Cloudinary media storage

    \item \textbf{External Services:}
    \begin{itemize}
        \item Payment gateways (Razorpay/Stripe)

\end{enumerate}

\section*{Key Features Implementation}

\subsection*{1. Hyperlocal Matching Algorithm}
The system implements a sophisticated matching algorithm that considers multiple factors:

\textbf{Algorithm Components:}
\begin{itemize}[leftmargin=*]
    \item \textbf{Distance Calculation:} Haversine formula for accurate geolocation distance
    \item \textbf{Availability Check:} Real-time worker availability verification
    \item \textbf{Rating Score:} Weighted average of user reviews
    \item \textbf{Completion Rate:} Historical service completion percentage
    \item \textbf{Response Time:} Average time to accept bookings
    \item \textbf{Specialization Match:} Exact match to required service category
\end{itemize}

\textbf{Scoring Formula:}
$$Score = (0.3 \times DistanceScore) + (0.25 \times RatingScore) + (0.2 \times AvailabilityScore)$$
$$+ (0.15 \times CompletionRate) + (0.1 \times ResponseTime)$$

\subsection*{2. Secure Payment Flow}
\begin{enumerate}[leftmargin=*]
    \item User initiates booking and payment
    \item Payment held in escrow state
    \item Worker accepts and completes service
    \item User confirms service completion
    \item Payment released to worker's account
    \item Automatic refund if service cancelled
\end{enumerate}

\subsection*{3. KYC Verification System}
\begin{itemize}[leftmargin=*]
    \item Government ID verification
    \item Address proof validation
    \item Police verification certificate (optional)
    \item Skill certification uploads
    \item Admin review and approval process
\end{itemize}

\subsection*{4. Rating and Review System}
\begin{itemize}[leftmargin=*]

\end{itemize}
